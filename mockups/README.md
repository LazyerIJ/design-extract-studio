# Peel — UIUX 리디자인 시안 (mockups)

design-extract-studio의 UIUX 리디자인 **제안 시안**입니다. 정적 HTML 프로토타입이라
브라우저에서 바로 열어볼 수 있고, 실제 앱 코드(`index.html`·`app.css`·`app.js`)는 아직
건드리지 않았습니다. 이 PR은 **방향 리뷰용**이에요.

## 파일
- **`C-mono.html` — ✅ 최종 제안 방향** (모노크롬 미니멀, Linear/Vercel 톤)
- `A-daylight.html` — 탐색안: 밝은 톤 + 인디고 (참고용)
- `B-midnight.html` — 탐색안: 다크 유지 + 코랄 (참고용)

## 보는 법
`C-mono.html`을 브라우저(Chrome 권장)로 엽니다. 정적이지만 주요 동작이 들어가 있습니다:

- **추출 실행** 클릭 → 주소창 잠김 + 진행 0→100% → 완료 시 결과·다운로드 표시
- 완료 후 **＋ 새 웹사이트 추출** → 주소창 다시 활성
- **옵션** 펼치기 / **고급 설정**(분석 범위·로딩 대기를 쉬운 말 선택지로)
- 결과의 **브레이크포인트 카드(Mobile/Tablet/Desktop)** 클릭 → 와이어프레임이 그 화면폭 구조로 전환
- **전부 다운로드**(체크박스 전체 선택) / 카테고리 칩으로 파일 필터
- **작업 이력** 카드 클릭 → 지난 작업 결과 다시 열기·다운로드

상태를 바로 보려면: `C-mono.html#idle` / `#running` / `#done`

## 반영된 결정
- 제품명 **Peel**(타이틀) + 라벨 "디자인 시스템 추출 스튜디오"
- 히어로: **"원하는 웹사이트를 넣으면, 디자인 시스템으로 정리해 드려요."**(볼드 포인트) + 처음 보는 사람용 설명
- **강조 입력 카드(검정)** — 가장 눈에 띄게, placeholder 안내, 상단 중복 버튼 제거
- 옵션 UX: 기본 접힘("추천 설정으로 진행돼요") + 추천 배지 + 어려운 숫자(크롤 깊이·SPA 대기)를 쉬운 선택지로 + 고급은 한 번 더 숨김
- 진행 상황: 단계 스테퍼 대신 **퍼센트 위주**
- 추출 상태 흐름: 대기 → 추출중(주소창 잠금) → 완료(＋새 추출로 해제)
- 다운로드: **주제별 분류**(핵심문서·색/토큰·레이아웃·모션·컴포넌트·이미지·AI/기타) — 레이아웃·모션이 파일 형식으로 흩어지지 않게. 파일 형식은 아이콘으로 표시
- **전부 다운로드** + 선택 다운로드 + **저장 폴더 선택**
- 결과: **브레이크포인트 카드 + 반응형 와이어프레임** + **`preview.html` 미리보기**(색·타이포·컴포넌트)
- 사이드바 제거 → **상단 슬림 바**
- **작업 이력에서 지난 작업 재다운로드**
